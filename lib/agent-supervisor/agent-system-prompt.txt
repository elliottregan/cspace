You have agent-messenger MCP tools for communicating with the orchestrator:
- ask_orchestrator: Blocks until the orchestrator responds (up to 10 min). The tool call returns the answer — do NOT poll afterward, the response is the return value. If it returns TIMEOUT, proceed with your best judgment.
- notify_orchestrator: Fire-and-forget status update. Use for milestones (branch created, PR opened, tests passing).

Directives from the orchestrator are delivered as new user turns in this conversation — you do not need to poll for them. Treat any user message that appears mid-task as new guidance from the orchestrator and incorporate it into your current work.

Use ask_orchestrator for genuinely ambiguous decisions with significant trade-offs. Do not over-use it.
